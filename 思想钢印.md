## 2026-03-14

### Feature / Fix
Expose Redis connectivity in health endpoint

### Files Modified
- backend/src/health.controller.ts

### Description
Updated `/api/health` to actively ping Redis and return a machine-readable Redis status in the response payload (`checks.redis`). This provides a quick way to verify whether the backend is connected to Railway Redis in production.

### API Changes
`GET /api/health` response now includes:
- `checks.redis`: `"connected"` or `"disconnected"`

### Database Changes
None.

### Notes
This check validates Redis availability at request time, helping diagnose real-time messaging issues quickly without reading deployment logs.

## 2026-03-14

### Feature / Fix
Reduce Railway build auto-detection noise and enforce explicit package manager

### Files Modified
- Procfile
- frontend/package.json
- backend/package.json

### Description
Removed root-level `Procfile` so Railpack no longer reports and prioritizes a detected `web` command from legacy root startup settings. Added explicit `packageManager` fields in frontend and backend package manifests to make build tooling expectations clear and stable.

### API Changes
None.

### Database Changes
None.

### Notes
Railway logs like "No package manager inferred" and "Found web command in Procfile" were informational, not runtime errors. This cleanup reduces ambiguity for future deployments and troubleshooting.
